(Fast) alles, was auf den Bildschirm kommt, basiert auf Vorlagen, das sind Abschnitte in HTML-Code, die das grundlegende Aussehen der darzustellenden Objekte festlegen.

Erst zur Laufzeit wird das entgültige Aussehen durch Setzen von Variablen bzw. Einsetzen von Information festgelegt.

Speicherort

Vorlagen können grundsätzlich in jedem Ordner abgelegt werden.
Standardmäßig liegen sie in einem Unterordner von templates.
Sie haben die Erweiterung ".tpl".

Aufbau

Vorlagen sind in Sektionen unterteilt. Diese sind durch eckige Klammern gekennzeichnet.

  • Jede Vorlage sollte eine Sektion [main] beinhalten.
  • Jede Sektion kann andere Sektionen mittels <!SEC:xy!> "aufrufen".

Kommentare

Für Kommentare wird "#" verwendet.
Soll "#" nicht als Kommentarzeichen interpretiert werden,
ist "\#" zu verwenden.

Vererbung

Jede Vorlage kann durch Einträge in der Sektion [include] auf bereits bestehende Vorlagen aufbauen. Dabei gilt:

  • Alle Sektionen der Eltern werden auf die Kinder vererbt (kumulativ).
  • Sektionen gleichen Namens werden von den Kindern überschrieben.
  • Konstante werden gleich nach dem Einlesen durch ihre Werte ersetzt!
  • Variablen und Wörterbuch-Einträge werden durch die Eltern ergänzt.

Variable

Variable können in der Sektion [vars] definiert werden.
Der Zugriff erfolgt mittels <!VAR:xy!>.

Sparchunterstützung

Sprachabhängige Info kann in den "[dic.??]"-Sektionen definiert werden. Dafür ist für jede Sprache eine Sektion anzulegen, z.B. [dic.en]. Auf diese Info kann mittels <!DIC:xy!> verwiesen werden. Den sprachspezifieschen Einträgen kann eine alltemeine Sektion [dic] folgen. Sie kommt zur Anwendung, wenn keine spezifischen Einträge vorliegen.

  • Der Aufruf von Sektionen erfolgt sprachunabhängig, d.h. statt <!SEC:xy.de!> wird nur <!SEC:xy!> geschrieben.
  • Wichtig: allgemeine Sprachdefinitionen müssen speziellen vorangehen!
  • Sprachspezifische Angaben können auch in den Dateien in design/dictionary gemacht werden.
