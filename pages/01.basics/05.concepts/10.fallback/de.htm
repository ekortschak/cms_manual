Generell wird (fast) jede Datei in folgenden Verzeichnissen gesucht:

APP_DIR
ist eine Konstante, die auf jenes Verzeichnis verweist, in dem index.php liegt. Sie wird automatisch angelegt.
APP_FBK

ist eine Konstante, die in der Konfigurationsdatei bestimmt werden kann. Sie kann auf jedes Verzeichnis am selben Server verweisen, auf die das php-Modul zugreifen kann.

Sie dient v.a. dazu, daß - bei mehreren Anwendungen am selben Server - nicht alle Dateien jedes Mal repliziert werden müssen. Stattdessen werden nur jene Dateien repliziert, die überarbeitet bzw. angepaßt werden müssen.

Dabei gilt:
  • Dateien in APP_DIR haben immer Vorrang!
  • Wird eine Datei in APP_DIR nicht gefunden, wird sie automatisch durch die gleichnamige Datei in APP_FBK ersetzt.
  • Die Datei-Struktur unterhalb beider Verzeichnisse muß identisch sein.

Beispiel

Am Server wurde ein Verzeichnis /var/www/cms/ angelegt. Dorthin wurden die CMS-Dateien nach dem Download entpackt und sollen nicht weiter verändert werden.

Die eigentliche Anwendung soll in /var/www/app/ liegen.
Dort wurde auch ein Ordner pages/home angelegt.

Folgende Dateien müssen nun nach /var/www/app/ kopiert werden:

  • index.php
  • x.*.php
  • config/*.*

sowie

  • Jede Datei, die nach den spezifischen Bedürfnissen der Anwendung überarbeitet werden muß, z.B. das Layout (design/layout/page.tpl).

Nun muß APP_FBK in config/asics.php definiert werden. Standardmäßig wird sie in einem übgeordneten Ordner gesucht.
